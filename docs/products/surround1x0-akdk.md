# Surround1x0-AKDK 商品ページ仕様

## 目的

`site/products/Surround1x0-AKDK/index.html` は、ホームと同じく 1 viewport を 1 メッセージとして扱う縦方向のセグメントページとする。実機の左右ユニットを常に体験の中心に置き、入力機能の統合、レイヤー設計、公開リポジトリへの導線を順番に伝える。

## セグメント構成

1. 左右ユニットを配置し、中央に「HIDの到達点」を表示する。
2. 左ユニットを左上かつ手前へ捻りながら退場させ、右ユニットを右側から手前へ大きく傾ける。左側の「キーボード、マウスカーソル、テンキーを統合。」は常にモデルより手前へ表示する。
3. 右ユニットを右上かつ手前へ捻りながら退場させ、左ユニットを左側から手前へ大きく傾ける。右側の「駆け抜ける入力。レイヤー切り替えのための頭脳。」は常にモデルより手前へ表示する。
4. 左右ユニットを再配置し、左にホームへ戻るリンク、右にGitHub リポジトリへのリンクを配置する。DOM順とキーボードフォーカス順も左から右へ一致させる。
5. ページ末尾のフッタを独立した設定セグメントとし、言語・テーマ設定を表示する。設定セグメントでは直前の `04` の3Dシーンを維持し、同一シーンへの移動は再発火させない。

デスクトップでは説明と実機を左右へ分ける。モバイルの第1セグメントでは説明を上、左右の実機を下へ配置し、両者の間隔を詰めたまとまりとして画面中央へ寄せる。第2・第3セグメントでは実機を上、説明を下へ移し、左右には寄せず画面中央へ配置する。モデルは画面幅のほぼ全域を使う大きさで表示し、上部の空間を埋める。いずれも指定メッセージの改行と主要リンクを 1 viewport 内に維持する。

## 3Dアセットと描画

- ライトテーマ: `site/products/Surround1x0-AKDK/assets/Surround1x0-AKDK-White.glb`
- ダークテーマ: `site/products/Surround1x0-AKDK/assets/Surround1x0-AKDK-Black.glb`

アクセントカラーは各モデルの球の方向性に合わせる。ダークテーマは赤（`#ff344a`）、ライトテーマはシルバーグレー（`#686d75`）をUIへ使用し、ナビゲーションの選択表示、ラベル、リンク、フォーカス表示、3Dモデルのリムライトを同系色で統一する。
共通テーマCSSが製品CSSより後に読み込まれる本番ビルドでも、製品固有のアクセント指定が共通のホーム用アクセントに上書きされないようにする。
- 描画: Three.js と `GLTFLoader`

各 GLB の `*_Left` と `*_Right` ルートノードを独立して移動し、モデル形状の加工や疑似アセットへの置換は行わない。通常のセグメント間移動は横方向の平行移動に見せず、行程の前半へ頂点を置いて浮き上がり、目標へは上方かつ手前側から降りる三次ベジェ曲線の弧で位置・奥行き・大きさを補間する。姿勢の補間には次の3層を重ねる。(1) 回転差分の約8%を行程60%付近で頂点にして戻すベル状オーバーシュート、(2) 移動量に比例して進行方向へ傾くピッチ・ヨー・ロールのバンク（その軸自体の姿勢変化が大きい移動では減衰させる）、(3) 退場と同じ対角軸を使う一過性のコーナーtwist（基準twistが大きく変化する移動では減衰させる）。所要時間は移動量に応じて伸縮させ、退場する側または移動量の大きい側のユニットを先行させて、もう一方をデスクトップ約110ms・モバイル約70msの遅延で追従させる。左右の移動量が等しい対称な移動では遅延を加えず対称性を維持する。再入場するユニットの透明度は行程の前半30%で不透明へ戻し、飛行中の姿を見せる。透明化が完了して駐機しているユニットの再入場は、遠い退場ポーズからではなく画面端付近のステージング姿勢から開始する（透明な状態での位置替えのため知覚されず、フェードインしながら降りるため、ビューポートのアスペクト比によらず早期に画面へ現れる）。再入場の前半は持ち上げの弧を使わず目標へ直進し、着地側のみ上方かつ手前からの弧で降りる。これにより退場側が画面から消える前に再入場側が画面内へ入り、セグメント移動中に左右ユニットが同時に画面から消えて何も表示されない瞬間を作らない。セグメント 2 / 3 間の退場は専用の三次ベジェ曲線で位置・奥行き・回転・大きさを補間する。いずれも `smootherstep` 系の曲線で開始・停止時の速度と加速度を0へ収束させ、スケールの終端オーバーシュートや跳ね返りは加えない。3Dは単一canvasへ描画して二重描画による合成アーティファクトを避け、全セグメントのコピー、リンク、案内テキストをそのcanvasより上のレイヤーへ置いて可読性を優先する。テーマ切り替え時は `document.documentElement.dataset.theme` を監視し、対応するモデルへ切り替える。

セグメント 2 / 3 間の退場は横方向の直線移動にしない。退場ユニットは画面内にいる前半で上昇、カメラへの前進、捻りを先行させ、後半は横滑りせずに画面上側の角（右ユニットは右上、左ユニットは左上）からカメラの脇を抜けるように退場する三次ベジェ軌道を通る。横方向の到達距離は縦方向と同程度まで抑え、奥行きの前進を主体としてこちらへ迫る印象を作る。移動中は縮小せず、カメラへ近づく動きとスケールの単調な拡大で画面を覆いながら抜ける（モバイルも縮小しない）。姿勢はモデルの奥側（画面上側）を `rotationX` で大きくこちらへ傾け、`rotationY` のヨーを退場側の側面（右ユニットは右側面、左ユニットは左側面）が見えるまで深く回頭させる（デスクトップ約-1.25rad、モバイル約-1rad）。この姿勢回転はtwistと同様に行程の前半約45%で開始・停止速度0のまま完了させ、側面が見える回頭を画面内で見せ切ってから軌道に乗って抜ける。再入場は同じ姿勢から正面へ巻き戻す。GLB上の右奥角はローカル`(+X, -Z)`、左奥角は`(-X, -Z)`にあるため、右は`(+X, +Z)`対角軸、左はその鏡像軸を使うQuaternion twistを補間する。このtwistは退場時間の前半42%で開始・停止速度を0に保ちながら完了させ、対象の奥角が画面端へ達するまでに中心よりカメラ側へ起き上がるようにする。右ユニットは右奥角、左ユニットは左奥角を手前へ起こしながら、それぞれ右上・左上へ退場する。左右はX位置、対角軸、回転角の符号を反転した鏡像とする。退場モデルの透明化は軌道の68%以降に開始し、斜め移動、前進、奥角の捻りが視認できる時間を確保する。入退場の透明度は左右ユニットごとに複製したThree.jsマテリアルへ反映し、反対側の非表示ユニットが完全不透明で一瞬描画されることを防ぐ。モバイルも距離を縮めた同じ方向性の軌道を使用する。

WebGL または GLB 読み込みに失敗した場合は、既存の `Surround1x0-AKDK.png` を静的フォールバックとして表示する。

第1セグメントでは左右ユニットを画面幅いっぱいに配置し、筐体前面よりキー面と天板が主に見える、真上に近い俯瞰角度へ傾ける。横方向の振り角と傾きは加えず、左右の上辺とキー列を同じ水平線へ平行に揃える。中央の「HIDの到達点」を挟む厳密な対称性を保ちつつ、タイトルの背後へ一部が回り込む大きさで、製品の横幅、入力面、工業製品としての精度を最初に印象づける。

第1セグメントの左右間隔は画面幅に追従させる。幅1065px以上では広い配置を維持し、672〜1065pxではモデルサイズを保ったまま左右のX位置を連続補間して中央へ寄せ、幅874px前後でも左右の筐体全体が画面内へ収まるようにする。672px以下はモバイル専用配置へ切り替える。

## 操作と検証

- 上部の `01`〜`04` と歯車の設定セグメント、マウスホイール、タッチ操作で各停止位置へ移動できる。
- URL hash は現在のセグメントと同期する。
- 既存の言語設定、テーマ設定、遷移元に応じた戻り先を維持する。
- `npm run test:surround-segments` で、本番ビルドのpreviewでは製品固有アクセントを検証し、開発サーバーでは4つのコンテンツとフッタ設定の5停止位置、テーマと GLB の対応、通常移動の三次ベジェ弧・バンク・一過性twist・先行/追従の公開状態と移動中の一過性twistの発生および収束、再入場ステージングの公開状態と退場・再入場の重なり（移動中に両ユニットが同時に画面から消えないこと）、デスクトップ／モバイルの横方向収まり、最終リンクと footer の表示を検証する。モバイルは標準的な縦長画面に加えて 338×619px と 446×619px でも、第1セグメントのコピーとモデルが間隔を詰めて中央へ収まり、第2・第3セグメントのモデルが中央かつ画面幅いっぱいに表示され、説明が切れないことを確認する。
