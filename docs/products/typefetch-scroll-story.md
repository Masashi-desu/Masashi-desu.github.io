# TypeFetch 操作説明スクロールストーリー

## 目的

TypeFetch 製品ページのヒーローデモと操作説明を連続した体験として見せ、スクロールだけで「呼び出す」「そのまま書く」「前面へ渡す」の関係を理解できるようにする。

## 表示仕様

- `#top` のヒーローは操作説明を含むストーリー領域内で viewport 背面に sticky 表示する。
- 操作説明中のアプリデモは画面サイズとスクロール位置にかかわらず viewport の中央へ固定し、レスポンシブな縮尺を保つ。
- ヒーローから操作説明へ入る瞬間は、ヒーローに表示されていた同じデモの画面上の矩形を始点として、viewport 中央の静止位置まで移動と縮小を同時に補間する。上方向へ戻る場合は、復帰を開始する瞬間のヒーロー内の実レイアウト位置を計測して終点に使い、ルール側からストーリーへ再進入した後でも画面外の古い座標へ移動したり、別位置からプレビューが再出現したりしない。
- 操作説明の先頭が viewport 中央の引き継ぎ線へ到達するまではヒーローデモを手動操作できる状態に保つ。中央を越えた時点で初めて固定表示へのトランジションとスクロール同期を開始し、浅いスクロールだけで操作を奪わない。
- 縦方向に余裕がない viewport では、操作説明を画面内へ入れる前にスクロール量へ同期してヒーローデモを上へ移動する。デモ全体が入る高さでは外枠まで viewport 内へ収め、入らない高さでは TypeFetch 入力パネルと確定操作を優先して表示してから、操作説明を上へ重ねる。上方向へ戻る場合はこの順序を逆再生する。
- TypeFetch ページの入場フェードではページ全体に `transform` を残さず、操作説明への切り替えが入場直後でも fixed デモの基準座標を viewport に保つ。
- 縦長の狭い viewport では、前面アプリのウィンドウを左右の余白を残して viewport 幅へ追従させる。ウィンドウバー、本文、文字、フッター操作はデスクトップより小さいモバイル用の縮尺にし、背景の前面アプリが TypeFetch 入力パネルより強いフォーカルポイントにならないようにする。TypeFetch 入力パネルは前面アプリと別の縮尺を使い、元の縦横比を維持する。
- `#manual`、`.tf-manual__header`、`.tf-move`、`.tf-move__body` は背景色を持たない透明な前景としてヒーローへ重なる。
- `.tf-rules` は同じストーリー領域に含め、半透明の前景として最終手順のアプリデモへ重ねる。デモの固定表示と最終状態はルールセクションの終端まで維持する。
- 操作説明が表示されている間はヒーローの見出し、リード文、スクロール案内を退避し、前景の説明と背面のアプリデモへ視線を集める。
- viewport 高さの 72% の位置を手順の切り替え線とし、キー表示がその線を越えた手順へ `.is-current` と `aria-current="step"` を付与する。
- 左カラムの見出しは上から `キーを押す。`、`書く。`、`渡す。` の順に並べ、current に対応する行だけ不透過、ほかの2行は半透明にする。
- 右カラムの手順本文とキー表示は右寄せにする。
- 手順番号用の縦レールは表示せず、中央のヒーローデモが見える面積を確保する。
- current ではない手順本文は透明度を下げる。

## デモ同期仕様

1. `01 / 呼び出す`
   - TypeFetch の入力ウィンドウを表示する。
   - 入力内容と前面アプリのテキストは初期状態にする。
2. `02 / そのまま書く`
   - 手順のスクロール進捗に合わせて、TypeFetch の入力欄へサンプル文を1文字ずつ反映する。
   - `prefers-reduced-motion: reduce` では文字単位の進捗演出を省略し、サンプル文を即時表示する。
3. `03 / 前面へ渡す`
   - サンプル文を前面アプリのキャレット位置へ挿入した状態へ切り替える。
   - TypeFetch の入力ウィンドウを閉じ、既存の挿入完了エフェクトを再生する。

日本語では `入力した文字を、前面へ。`、英語では `The text moves to the frontmost app.` をサンプル文に使う。

## 操作とアクセシビリティ

- 操作説明の先頭が viewport 中央へ到達するまではヒーローを操作可能に保つ。中央を越えてスクロールストーリーへ引き継いだ後だけヒーローを `inert` とし、透明な前景越しに背面のフォームへフォーカスやクリックが移らないようにする。
- スクロールに伴う状態変更はデモンストレーションであり、ライブリージョンによる連続読み上げは行わない。
- ヒーローへ戻ったときは、ストーリー開始前の入力値、選択位置、TypeFetch の開閉状態、ステータスを復元し、従来の手動デモをそのまま操作できるようにする。
- 操作説明より下へ進んだ場合は最終状態を保ち、上方向へ戻ると同じスクロール位置から3段階を可逆的に再現する。
- ルールセクションを通過する間も最終状態を保持し、ルールセクションの終端を越えた時点で固定デモを終了する。
