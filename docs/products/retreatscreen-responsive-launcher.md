# RetreatScreen ランチャーのレスポンシブ仕様

RetreatScreen 製品ページ先頭のランチャーパネルは、表示幅が `48rem` 以下のスマホ向けレイアウトでは初回表示時の可視 viewport 高に合わせ、その高さを表示中は安定して保つ。iOS の実機内蔵ブラウザでは `100svh` が上下のブラウザ UI を除いた可視領域より大きくなる場合があるため、初回の `visualViewport.height` を同期した CSS 変数を優先し、取得できない環境だけ `100svh` を使う。

- 通常のスマホ縦持ちでは、パネルを画面高まで伸ばす。
- スクロールに伴ってブラウザバーが開閉してもパネルを伸縮させない。同じレイアウト幅で発生する `visualViewport` の高さ変更は無視し、端末の向き変更、表示幅変更、履歴復帰時だけ高さを再計測する。
- アイコン一覧は、本家の上から並ぶ構成を尊重し、タイトル／Edit ボタンを含むヘッダー下端から約40〜50pxの余白を取って上詰めする。画面が高くなっても一覧を縦中央へ移動しない。
- 幅と高さの比が `9:20` 以下（高さが幅の約 2.22 倍以上）の極端な縦長ブラウザでは、パネル高を本家 Mac アプリの `796 × 850` 比率に制限する。通常のiPhone 17 Pro相当の比率は画面高への追従を維持する。
- `48rem` を超え、本家比率を維持できる表示では、タイトル、閉じるボタン、Edit ボタン、アプリアイコン、セル、グリッドを本家寸法から一様に拡縮する。

この境界は `tests/playwright/retreatscreen-mobile-panel-fit.js` で Chromium と WebKit の両方を検証する。レイアウト viewport より `visualViewport` が低い iOS 内蔵ブラウザ相当の状態、ブラウザバー開閉時の高さ固定、向き変更後の再計測も同テストで再現する。

## 編集モードの操作仕様

- 通常モードでは各アイコンをリンクとして操作でき、ホバー時の押下可能な視覚効果を表示する。
- 編集モードでは各アイコンの `href`、ページ遷移、通常時の押下可能な視覚効果を無効にし、並べ替え可能な編集ボタンとして扱う。アイコン一覧内に名前入力欄は表示しない。
- ページが2枚以上ある場合は、ランチャーパネル内で編集状態にかかわらず同じように横スクロールを検出し、本家同様にトラックパッドやマウスで前後のページへ移動できる。アイコン編集ウィンドウの表示中とアイコンのドラッグ中は、意図しないページ移動を防ぐため横スクロールを受け付けない。
- 編集モード中にアイコンを押すか、キーボードの Enter / Space で実行すると、本家同様のアイコン編集ウィンドウを開く。ウィンドウ表示中は背面のランチャーを `inert` にして操作対象から外し、Esc、左上の×、キャンセルのいずれでも下書きを破棄して閉じる。
- アイコン編集ウィンドウでは表示名とカスタム画像を下書きとして扱い、保存時だけ一覧へ反映する。画像は PNG / JPG / GIF / SVG を受け付け、PNG / JPG / SVG は 384px 四方の透過 PNG へ収め、GIF はアニメーションを保持する。
- 「元の参照アイコンに戻す」は画像だけを初期状態へ戻す下書き操作とし、保存で確定、キャンセルで破棄する。保存した表示名と画像は `localStorage` に保持し、再読み込み後も復元する。
- 編集モードを終了すると、各アイコンのリンク先、ページ遷移方向、キーボードフォーカス、押下可能な視覚効果を通常モードの状態へ戻す。
- 編集中に言語を切り替えて戻り先リンクが再同期されても、編集モードを終了するまではアイコンのリンク操作を再有効化しない。

この操作境界は `tests/playwright/retreatscreen-edit-mode-actions.js` で Chromium と WebKit の両方を検証する。
